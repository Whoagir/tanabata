# Краткое описание игры и список механик

## Описание игры

**Tanabata** — 2D Tower Defense на гексагональной карте. Цикл: строительство (до 5 башен) → выбор 2 башен для сохранения → волна врагов. Башни питаются от энергосети майнеров на руде. Цель — не дать врагам дойти до выхода. Волны 1–39, крафт спецбашен по рецептам.

---

## Список механик

### Фазы
- Фаза **BUILD** — строительство.
- Фаза **SELECTION** — выбор башен для сохранения.
- Фаза **WAVE** — волна врагов.
- Переход BUILD → SELECTION → WAVE → BUILD (авто или клик по индикатору).
- Ручной переход на следующую фазу по клику на индикатор фазы.

### Размещение и снятие башен
- Размещение башни по ЛКМ на проходимый гекс с возможностью размещения.
- Снятие башни по ПКМ.
- Лимит: до 5 «обычных» башен за фазу BUILD (дебаг-башни не в счёт).
- После 5 башен — автоматический переход в SELECTION.
- Майнер можно ставить только на гекс с рудой.

### Выбор башен (SELECTION)
- Игрок отмечает 2 башни для сохранения (клик по башне).
- Невыбранные временные башни превращаются в стены.
- Майнеры считаются выбранными по умолчанию.
- Клик по индикатору фазы — переход в WAVE (финализация выбора).

### Типы башен
- **ATTACK** — атакующая башня (стреляет по врагам, тратит руду).
- **MINER** — майнер на руде, источник энергии в сети.
- **BATTERY** — батарея: режим Добыча (жила под батареей питает сеть, батарея заряжается из сети) или Трата (запас отдаётся сети как источник руды). Крафт: NA2+NU2+TOWER_MINER.
- **WALL** — стена, блокирует путь, не участвует в энергосети.

### Типы атак башен
- **PROJECTILE** — снаряд (полёт к цели, потиковая донаводка).
- **LASER** — мгновенный луч (опционально с замедлением).
- **AREA_OF_EFFECT** — Volcano, урон по области.
- **BEACON** — Lighthouse, вращающийся луч по секторам.
- **NONE** — нет атаки (аура, поддержка).

### Типы урона
- **PHYSICAL** — снижается физ. бронёй врага.
- **MAGICAL** — снижается маг. бронёй врага.
- **PURE** — игнорирует броню.
- **SPLIT** — половина урона как PHYSICAL, половина как PURE (Хьюдж).
- **SLOW** — наложение замедления (движение).
- **POISON** — урон по времени (обычный яд).
- **INTERNAL** — внутренний (ауры и т.п.).

### Параметры атаки (params)
- **split_count** — число снарядов на выстрел (несколько целей).
- **slow_multiplier / slow_duration** — параметры замедления (лазер: Silver 30%, Silver Knight 40%, Huge 60%).
- **damage_split** — для SPLIT: доли типов урона (например PHYSICAL 0.5, PURE 0.5).
- **crit_chance / crit_mult** — шанс и множитель крита (лазер: Pink 35% x2.6, Huge 10% x9).
- **crit_splash_radius_hex / crit_splash_factor** — при крите урон по площади (Huge: 0.7 гекс, 30% урона).
- **impact_burst** — при попадании дополнительные снаряды (fragment_count, radius_hex; опционально fixed_damage, fragment_speed_multiplier). Malachite: homing осколки; U235: fixed_damage, fragment_speed_multiplier.
- **projectile_speed_multiplier** — множитель скорости снаряда (238, U235 и др.).
- **repeat_hit_bonus_damage / repeat_hit_stack_duration** — U235: бонус урона за стак повторных попаданий по одной цели, длительность стака в секундах (стаки в game_state["tower_hit_stacks"], декремент в StatusEffectSystem).
- **effect: JADE_POISON** — Jade: стакающийся яд (снижение регена).
- **bash_chance / bash_duration** — Изумруд: шанс обездвижить врага на время.

### Ауры башен
- Аура **speed_multiplier** — ускорение стрельбы союзных башен в радиусе (DE, Silver и др.).
- Аура **damage_bonus** — бонус урона союзным башням в радиусе (при наличии в данных).
- Учёт ауры в CombatSystem (fire_rate), расход руды за выстрел с коэффициентом для аур (Config).

### Энергосеть
- **Вышки типа Б:** MINER и BATTERY. Батареи не выпадают с волны, у них crafting_level 1.
- Связи между башнями: MST по приоритетам (майнер–майнер и связи с батареей > майнер–атакер > атакер–атакер).
- Майнер–майнер, майнер–батарея, батарея–батарея: до 4 (майнер) или 5 (батарея) гексов по одной линии (на одной прямой), без другой вышки типа Б между ними.
- Соседи (1 гекс): любые пары майнер/батарея/атакер.
- Источники руды для потребителей: жилы под майнерами и батареями + батареи в режиме разряда (запас). Списание через get_power_source_reserve / consume_from_power_source.
- Атакующая башня стреляет только при доступе к руде через сеть (от майнеров и/или батарей).
- **Сопротивление (resistance):** по пути до ближайшей вышки типа Б каждая атакующая вышка даёт +0.2 (кап 1). Множитель урона и силы аур: max(0, 1 - resistance). Аура от вышки с resistance 0.4 даёт, например, 20%×0.6 = 12% бонуса.
- **Вкл/выкл башни вручную:** ПКМ по башне (WAVE/SELECTION) или слайдер в InfoPanel (50 px правее названия). Выключенная башня не стреляет и не тратит руду, но остаётся в сети; при включении пересчёт без полной пересборки. Выключенный майнер только передаёт сеть, не добывает и не списывает руду с жилы.
- Пересборка сети при постановке/снятии башни и смене фазы.
- Майнер на линии между двумя майнерами — «перехват» (старая линия разрывается, две новые).

### Формула урона
Урон по врагу: **base_damage × network_ore_mult × mvp_mult × resistance_mult × early_mult** плюс карты и бонус ауры урона. network_ore_mult — от доли руды в сети (1.0–1.5); mvp_mult = 1 + mvp_level×0.2; resistance_mult = max(0, 1 - resistance); early_mult — от раннего крафта (проклятие).

### Руда
- Процедурная генерация руды при старте карты (несколько жил, общая мощность в лимите). Центральная жила — один множитель мощности (CENTRAL_VEIN_POWER_MULT). Дополнительная точка руды: на серединном перпендикуляре между центральной и средней жилой, на расстоянии 3 гекса от середины отрезка, 15–20 руды (одна сущность ore).
- У каждого майнера — связь с гексом руды (ore): запас current_reserve, max_reserve.
- **Восстановление руды за волну** — фиксированное: во время волны до 20 тиков (1 тик в секунду реального времени); за волну на жилу с майнером восстанавливается ровно restore_per_round × множитель по номеру волны (длинные волны не дают лишней руды). При короткой волне в конце добавляется остаток. Множитель: Config.get_ore_restore_mult_for_wave(wave_number). **База восстановления по уровню майнера (1–5):** 1.7, 2.5, 5.2, 9.6, 15.0 (руды за волну до глобального множителя и прогресса уровня). **Выключенный майнер** даёт на своей жиле +30% к восстановлению. **Бонус отклонения успеха:** восстановление умножается на Config.get_success_ore_bonus_mult(success_level) — см. ниже.
- **Батарея** в режиме «добыча»: в хранилище добавляется **charge_rate** руды в секунду (в данных 0.5); эта величина тоже умножается на бонус отклонения успеха.
- Выстрел атакующей башни стоит **shot_cost** руды; списание с сети майнеров. Расход учитывается по секторам (центр/середина/край) для логов.
- Руда ниже порога — «истощена», майнер неактивен.
- HUD: индикатор руды — по одному прогресс-бару на каждую сеть с рудой (при старте без вышек баров нет); под каждым баром подпись «X / Y»; синий при руде >= 10, красный при < 10, мигание при < 3. При наведении на майнер — руда его сети. **Основная сеть** — сеть с максимальным числом атакующих вышек; в отчёте после волны выводится остаток руды в основной сети в %.

### Крафт
- Книга рецептов — клавиша **B**.
- Крафт в фазе SELECTION по подсвеченной группе руды (рецепт выбирается в книге).
- Рецепты заданы в recipes.json: входные башни/руда → выходная башня.
- Реализованные рецепты: Silver, Malachite, Volcano, Lighthouse, Jade, Изумруд (DA2+TE1+NA1), Парайба (TOWER_GRUSS+NE1+NI2) и др. (полный список в recipes.json).
- Урон башен в данных масштабируется по ярусу крафта (scripts/balance_tiers.py): crafting_level 0 — −15%, 1 — +7%, 2 — +10%. HP врагов при спавне (WaveSystem): множители из wave_balance.json (волны 1–3 ×0.6; 4 +40%; 5 ×0.75; 6–7 +15%/ +25%; 8 ×1.03125 для быстрого; 9 +20%; 11 ×0.76; 14 ×0.8; 15 ×1.5; 16 ×1.7; 21–22 ×3.0; 27 ×0.8; 37–39 ×3/×3/×4); дополнительно быстрые (ENEMY_FAST) на волнах 6–9: Config.ENEMY_FAST_WAVES_6_9_HP_MULT (0.6). В туториале — ×0.3.

### Крафт с касания
**«С касания»** — это крафт, который можно сделать в фазе выбора (SELECTION), потому что в фазе строительства выпали башни, из которых собирается рецепт. Игрок не получает «дроп» от клика по башне: он ставит пять башен в BUILD, переходит в SELECTION и там крафтит из того набора, что выпал. Пример: выпали майнер, TE1, TO1 — если есть ещё TA1 (или TA2), можно скрафтить Auriga (TA+TE+TO). Проклятие касания (−15% урона) вешается только если результат крафта — вышка **уровня крафта 1 и выше** (Auriga, Silver, Malachite и т.д.); апгрейды базы (TO1+TO1=TO2, PO1+PO1=PO2) — без проклятия. Батарея проклятие касания не получает. Снять можно благословением «Снять проклятия».

**Первая волна (без туториала) — особые правила:**
- **Слот 0:** всегда **майнер** (TOWER_MINER).
- **Слоты 1–4:** выпадают ровно в формате **Б А1 А1 А2 А3** — одна атакующая база (А1) повторяется дважды, две другие (А2, А3) по разу; порядок слотов случайный (например Б А1 А1 А2 А3, Б А1 А2 А1 А3, Б А1 А2 А3 А1 и т.д.).
- Набор из трёх баз (А1, А2, А3) выбирается так, чтобы **не было крафта с касания**: не выдаются все три базы под Auriga (TA+TE+TO) и не все три под Malachite (PA+PE+PO). То есть всегда 2 базы из одной группы + 1 из другой (TA/TE/TO vs PA/PE/PO).
- Итог: на первом уровне **крафта с касания нет** — из выпавших пяти башен нельзя собрать ни один рецепт.

### Перемешивание волн по интервалам (реиграбельность)
- Внутри фиксированных **промежутков** волны рандомно меняются местами; порядок контента сохраняет баланс по **общему эффективному HP** (EHP). Интервалы: **6–9**, **11–15**, **16–19**, **21–29**, **31–39** (боссы 10, 20, 30, 40 не участвуют).
- При старте партии (не в туториале) для каждого интервала: считают EHP каждой волны (формулы как в scripts/wave_hp_table.py: броня, способности, реген, путь, скорость), перемешивают порядок волн внутри интервала, затем для каждой «слота» задают множитель **health_scale** так, чтобы EHP слота совпадал с исходным EHP этого слота. Множитель применяется только к **HP** врагов; реген считается отдельно (таблица по волнам + regen_scale от пути/скорости) и на health_scale не умножается. В итоге игрок видит другой порядок врагов, но кривая сложности по EHP сохраняется.
- Реализация: **WaveEffectiveHP.compute_wave_effective_hp()** (core/wave_effective_hp.gd), **GameManager._build_wave_shuffle_map()**, **game_state["wave_shuffle_map"]** (slot_wn -> source_wave_number, health_scale); в WaveSystem при спавне врага подставляется определение source-волны, HP умножается на health_scale, реген — из Config (таблица + regen_scale).

### Волны
- Волны 1–39 заданы в waves.json (или константы/инициализация в wave_system).
- Параметры волны: враг(ы), количество, spawn_interval, health_multiplier / health_override, abilities, evasion_chance. Реген врагов задаётся таблицей по волнам в Config и regen_scale (длина пути, скорость, способности); в waves.json только **regen_multiplier_modifier** для точечной подстройки.
- **Модификаторы волны** в JSON: health_multiplier_modifier, speed_multiplier_modifier, regen_multiplier_modifier, magical_armor_multiplier и др. — умножаются на базовые значения при спавне.
- Дополнительные множители HP заданы в **data/wave_balance.json** (wave_health): диапазоны 1–3 и 5–6, ранние 1–5, волны 4–9, 11–12, 14–16, 21–22, 27, 37–39 (см. файл). Волна 8: −25% только для ENEMY_FAST; быстрые на волнах 6–9 дополнительно ×0.6 (Config.ENEMY_FAST_WAVES_6_9_HP_MULT). WaveSystem применяет через DataRepository.get_wave_health_code_multiplier(). Итоговые множители — в _final_multipliers.
- Смешанные волны: массив enemies[] с разными типами и долями.
- Отдельные множители HP для летающих и наземных (health_multiplier_flying / health_multiplier_ground).
- Спавн по таймеру (каждые spawn_interval сек один враг).
- Завершение волны: все заспавнены и **alive_enemies_count == 0** (счётчик уменьшается при убийстве и при достижении выхода) → лог отчёта (log_wave_damage_report), переход в BUILD.

### Враги (типы)
- Слабый 1/2, Обычный 1/2, Крепкий 1/2.
- Магический щит, Физический щит.
- Быстрый 1/2.
- Босс.
- Летающий, Летающий слабый/крепкий/быстрый.
- Хиллер (аура лечения), Танк (агрро).
- Параметры: health, speed, physical_armor, magical_armor, flying. Множители HP по волнам — в wave_balance.json; плюс модификаторы волны из waves.json и сложность (difficulty).

### Голд-существо
- При появлении у каждого врага (кроме босса) с вероятностью **1/400** он может стать «голдом». Ограничения: уровень успеха не ниже 10; за волну может появиться не более одного голд-существа.
- Голд-существо: **HP в 15 раз больше** базового для этого типа/волны, урон игроку не меняется. Визуал — золотой цвет.
- За убийство голд-существа игрок получает **+70 руды**, распределённых равномерно по всем жилам в сетях (под майнерами). Константы: Config.GOLD_CREATURE_CHANCE, GOLD_CREATURE_HP_MULT, GOLD_CREATURE_ORE_REWARD, GOLD_MIN_SUCCESS_LEVEL.

### Способности врагов (пассивные)
- **effect_immunity** — иммунитет к эффектам (slow, poison, слой и т.д.).
- **evasion** — шанс увернуться от попадания (evasion_chance на волне).
- **reactive_armor** — стаки брони и регена от полученных ударов.
- **kraken_shell** — сброс дебаффов при достижении порога полученного урона.
- **untouchable** — попавшая по врагу башня получает замедление.
- **hus** — реген до 250% при низком HP.
- **healer_aura** — +20 регена союзникам в радиусе 4 гекса (только у Хиллера).

### Способности врагов (активные)
- **rush** — ускорение ×2.3 на 6 с (RUSH_DURATION), кулдаун 5 с.
- **disarm** — башни в радиусе не стреляют (на время).
- **blink** — телепорт вперёд по пути. По умолчанию (Config): 6 гексов, кулдаун 7.2 с, старт 5.2 с. На волне 14 заданы 4 гекса, кулдаун 8.2 с, старт 6.2 с; при спавне добавляется случайный сдвиг 0–0.2 с к начальному кулдауну.
- **reflection** — 4 слоя щита, каждый удар снимает 1 слой, кулдаун 5 с (отражает урон/эффекты пока есть слои).
- **aggro** — Танк: башни в радиусе 4 гекса бьют только этого врага, 2 с, кулдаун 5 с.

### Путь и движение врагов
- Путь: A* между чекпоинтами (Entry → Checkpoint 1…6 → Exit).
- Пересчёт пути при постановке/снятии башни (дебаунс).
- Летающие враги: условная вставка центра карты в путь (длинные отрезки, вход/выход).
- Движение по path.hexes с текущим индексом; к следующему чекпоинту при приближении. При входе на гекс применяется **экологический урон**: руда (фикс. ENV_DAMAGE_ORE_PER_TICK при проходе по жиле выше порога), энерголиния — урон max(1, руда_в_сети × уровень_игрока / 40) (в 8 раз ниже прежней формулы).
- Эффективная скорость с учётом **slow** (множитель).
- **Bash** — обездвиживание: скорость = 0, блокируются рывок и блинк.
- Дойдя до выхода — урон игроку, сущность врага удаляется. Итоговый урон игроку: база волны × множитель длины пути (Config.get_path_length_enemy_damage_to_player_mult) × **Config.ENEMY_DAMAGE_TO_PLAYER_GLOBAL_MULT** (по умолчанию 0.8, т.е. −20%).

### Выбор цели (CombatSystem)
- Цель: живой враг в радиусе (range) башни, сортировка по расстоянию, до split_count целей.
- **Агрро**: если у какого-то врага активен aggro и башня в радиусе 4 от него — цель только этот враг (если в range).
- Энергия: проверка доступа к руде через сеть, запас >= shot_cost; иначе выстрел не производится.

### Урон и броня
- Формула снижения урона броней (физ/маг в зависимости от damage_type).
- **PURE-урон** снижается сопротивлением из волны (pure_damage_resistance в waves.json, записывается во врага при спавне); итоговый урон × (1 − get_pure_damage_resistance). Применяется в projectile, combat, beacon, volcano, auriga.
- **Evasion**: перед применением урона проверка roll_evasion; при успехе — промах (0 урона, без эффектов).
- Рефлекшн: при наличии слоёв reflection часть попаданий снимает слой и не наносит урон/эффект (в т.ч. яд).
- **Проклятие curse_hp_percent**: дополнительный расход руды за выстрел +0.5 (get_curse_extra_ore_per_shot); учитывается в CombatSystem, Auriga, Volcano, Beacon.

### Снаряды (ProjectileSystem)
- Полёт к цели, обновление позиции каждый кадр.
- Потиковая донаводка: раз в HOMING_TICK_INTERVAL коррекция направления к цели (если в радиусе активации).
- Полный homing для осколков Malachite (impact_burst).
- При достижении цели: применение урона (с броней, evasion, reflection), наложение эффектов (slow, poison, JADE_POISON, bash), damage_flash на враге.
- Лазер: мгновенный урон по линии, без сущности снаряда; длительность отрисовки из Config.

### Статус-эффекты на врагах
- **Slow** — множитель скорости движения < 1 на время (Silver и др.).
- **Poison** — урон по времени (тик), длительность. **Стакается по источнику (def_id башни):** на одном враге могут действовать одновременно яд от NU1, NU2, Либра и т.д.; повторное попадание от той же башни (тот же def_id) только обновляет таймер и dps. Jade (JADE_POISON) хранится отдельно и стакается по количеству попаданий.
- **Jade poison** — стакающийся дебафф, снижение регена по стакам (Jade).
- **Bash** — обездвиживание на время, блокирует движение, рывок, блинк (Изумруд).
- Рефлекшн снимает один слой за попадание (в т.ч. от яда/эффектов).

### Volcano и Beacon
- **Volcano** (AREA_OF_EFFECT): периодический урон по области вокруг башни (VolcanoSystem).
- **Lighthouse** (BEACON): вращающийся луч, урон врагам в текущем секторе (BeaconSystem, beacon_sectors).

### MVP
- За волну ведётся учёт урона по башням (tower_damage_this_wave).
- По завершении волны (все враги убиты, не пропущена): башня с максимальным уроном получает +1 MVP (если у неё mvp_level < 5).
- MVP уровень 0–5 даёт множитель урона башни.
- При ручном переходе WAVE → BUILD (пропуск волны) MVP за эту волну не начисляется.
- HUD: топ-5 башен по урону с отображением MVP и подсветкой первой.

### Очки (счёт) и бонус отклонения успеха
- **Счёт = суммарный опыт (XP).** За каждое убийство врага даётся **Config.XP_PER_KILL** (10) очков, умноженных на **бонус отклонения успеха**: Config.get_success_ore_bonus_mult(success_level). Стандарт успеха — 10; при отклонении 2 (успех 8 или 12) множитель 1.1 (+10%), при 4 — 1.2, при 6 — 1.3 и т.д. без ограничения (5% за единицу отклонения). Тот же множитель применяется к восстановлению руды майнерами и к добыче батареи (charge_rate).
- На экране Game Over показывается упрощённый «суммарный XP» (текущий XP в уровне + по 100 за каждый уже полученный уровень). Точный суммарный XP для аналитики: Config.get_total_xp(level, current_xp) с учётом порогов уровней (580, 1850, 2070…). Макс. уровень игрока — 5.

### Проигрыш и конец игры
- При **здоровье игрока 0 или ниже** (если не включён режим разработчика) игра заканчивается: **game_state["game_over"] = true**, показывается модальное меню с очками (XP), числом убийств и кнопками «Начать заново», «В меню». Фон меню — тёмно-синий, как в главном меню. Ввод под меню не проходит.
- В **режиме разработчика (I)** при HP <= 0 игра не заканчивается (Config.god_mode).

### UI и управление
- Индикатор фазы (круг): синий BUILD, жёлтый SELECTION, красный WAVE. **Клик по индикатору переключает фазу только в режиме разработчика (I)**; в обычном режиме индикатор не кликабелен.
- Смена фазы — клик по индикатору фазы (только в режиме разработчика I) или авто-переходы (5 башен → SELECTION, 2 выбранных → WAVE, конец волны → BUILD).
- Клавиша **B** — книга рецептов.
- Клавиша **P** — пауза.
- Кнопки скорости времени: 1x, 2x, 4x.
- Здоровье игрока, номер волны, счётчик башен, убийства.
- InfoPanel: при выборе башни/врага/руды — детали (урон, способности и т.д.). Для башни (не стена): переключатель вкл/выкл в заголовке (50 px правее названия), слайдер в стиле «замок в самолёте» (приглушённые зелёный/красный).
- Клики по карте обрабатывает InputSystem; клики по UI не передаются в мир (в т.ч. при открытом попапе туториала).

### Дебаг / режим разработчика
- Клавиша **I** — вкл/выкл **режим разработчика**: дебаг-метки (координаты гексов при наведении), панель волны, **кликабельный индикатор фазы**, **игрок не проигрывает при HP <= 0** (god_mode).
- Панель волны: выбор номера волны (0–40) для следующего старта; 0 — идти по счётчику.
- Клавиши **1 / 2 / 3** — тип башни при размещении в обход лимита: 1 — случайная атакующая, 2 — майнер, 3 — стена.
- Клавиша **6** — тестовая башня из списка (Config.DEBUG_TEST_TOWER_IDS: TOWER_QUARTZ, NI3, TA2, TOWER_LAKICHINZ); повторное 6 выключает.
- Клавиша **0** — выключить любой дебаг-тип башни.
- **Shift+PageUp** — профайлер (если включён).

### Уровни и обучение (LevelConfig)
- **Основная игра** — конфиг без лимита волн, 6 чекпоинтов, без пошаговых подсказок.
- **Туториал** — уровни 0–4: Основы (пустая карта, фазы, майнер на руду), Энергия и руда (гиперболизированная трата), Стены и выбор, Крафт, Практика (10 волн). Параметры: map_radius, ore_vein_count, wave_max, checkpoint_count (0 = без чекпоинтов), steps (триггер + сообщение). Триггеры: towers_5, phase_selection, miner_on_ore, miners_2_on_ore, wave_started и др.

### Логирование и аналитика
- После каждой завершённой волны в консоль выводится **отчёт**: длительность, чекпоинты (%), путь (гексы, макс/ср %), руда по секторам (до/после/восстановлено), траты по секторам, основная сеть (остаток руды в %), башни по def_id, урон вышек за волну и всего.
- **Сводная таблица для графика** — таб-разделённые колонки (волна, длит_игр, путь_гексов, руда_трат, руда_сек, до_0…до_5, времена сегментов, hp_игрока, lvl_игрока, xp_всего, пропущено_врагов, руда_всего, **успех**). Колонка успех: 1 — волна пройдена (HP > 0), 0 — проигрыш. Скрипт **scripts/plot_wave_logs.py** (matplotlib, numpy) парсит таблицу и строит графики (длительность, путь, руда, чекпоинты, дашборд); поддерживает ведущую строку [Сводка волн для графика].

### Таблица HP по волнам
- В **data/WAVE_HP_TABLE.md** — суммарное сырое HP, реген/с, EHP по типам урона, скорость, mixed EHP и общее эффективное HP по волнам (с учётом способностей, регена за 10 с, летающих и длины лабиринта 150–900). Скрипт **scripts/wave_hp_table.py** пересчитывает таблицу и графики (WAVE_EFFECTIVE_HP.png, производная). Запуск: `wsl python3 scripts/wave_hp_table.py` (или `py -3`).
