# Логирование и аналитика волн

## Назначение

После каждой завершённой волны GameManager выводит в консоль детальный отчёт и сводную таблицу для последующего анализа и построения графиков.

## Когда пишется лог

**log_wave_damage_report()** вызывается в WaveSystem при завершении волны: все враги заспавнены и ни одного живого не осталось. При пропуске волны (ручной переход WAVE→BUILD) отчёт не выводится.

## Содержимое отчёта (консоль)

- **Лабиринт:** длительность волны в игровом и реальном времени, число врагов в волне.
- **Чекпоинты:** сколько врагов дошло до каждого чекпоинта (0…max_cp), в процентах от волны и среднее за игру.
- **Среднее время между чекпоинтами** (игровое время) по сегментам пути (0→1, 1→2, …).
- **Путь:** длина пути в гексах, максимальный и средний % пройденного пути врагами, средние за игру.
- **Руда по секторам:** центр / середина / край — до и после волны, восстановлено за волну, число майнеров.
- **Майнеров всего**, **руда:** израсходовано за раунд, руда/сек (среднее), добыто за волну, гексов занято майнерами.
- **Руда траты по секторам:** центр, середина, конец.
- **Основная сеть:** (если есть атакующие башни) остаток руды в %, текущ/макс, израсходовано в %. Под «основной сетью» понимается сеть с максимальным числом атакующих вышек (get_main_network_ore_stats).
- **Башен всего** и группировка по def_id (например «TOWER_WALL x30, PA1 x2»).
- **Урон вышек за волну** и **Урон вышек всего** (накопительно со средним за раунд).

## Сводная таблица для графика

В конец лога выводится блок **[Сводка волн для графика]** — таблица с табуляцией в качестве разделителя. Колонки: волна, длит_игр, длит_реал, врагов, путь_гексов, путь_макс%, путь_ср%, руда_трат, руда_сек, руда_доб, майнер_гексов, руда_центр, руда_серед, руда_конец, до_0…до_5 (процент дошедших до чекпоинтов), 0->1, 1->2, … (время между чекпоинтами по сегментам), hp_игрока, lvl_игрока, xp_всего, пропущено_врагов, руда_всего, успех (уровень успеха, число 6+, в игре в среднем 10 — не 1/0).

После сводной таблицы может выводиться блок **[Руда по вышкам за раунд]** с подзаголовком (игр. время раунда в секундах), затем строка-заголовок: вышка, def_id, руда_всего, руда_сек — и строки по одной вышке (среднее потребление руды за раунд и руда/сек).

Оба блока можно скопировать в один файл и передать plot_wave_logs.py для построения всех графиков.

## Скрипт plot_wave_logs.py

**Путь:** `scripts/plot_wave_logs.py`

**Назначение:** парсит сводную таблицу (заголовок TABLE_HEADER и строки TABLE_ROWS в скрипте или данные из лога) и строит графики.

**Зависимости:** matplotlib, numpy (`pip install matplotlib numpy`).

**Запуск:** `python plot_wave_logs.py [файл]` (например `wave_log_latest.txt`). Под WSL: `wsl python3 plot_wave_logs.py wave_log_latest.txt` из папки scripts. Скрипт пропускает ведущие строки до заголовка (например [Сводка волн для графика]); колонка успех опциональна (уровень успеха; для старых логов подставляется 10).

**Графики:** длительность волн, размер волны и путь, чекпоинты (прохождение в %), время между чекпоинтами по сегментам, **уровень успеха по волнам** (wave_success.png), **руда по вышкам** (tower_ore_per_round.png — барчарт по таблице [Руда по вышкам за раунд], если блок передан в файле или задан TOWER_ORE_ROWS), тепловая карта чекпоинтов, метрики руды (траты, руда/сек, добыча, майнер-гексы, руда по секторам), дашборд по ключевым показателям. Используется для баланса и анализа проходимости волн.

**Формат данных:** скрипт ожидает колонки как в TABLE_HEADER; при неполном наборе колонок (старый формат лога) делается подстановка нулей для недостающих полей (путь_гексов, руда_*).

## Папка scripts и визуализация данных

Папка **scripts/** в корне проекта содержит скрипты для баланса и визуализации игровых данных. Это важно для анализа прогонов и баланса на будущее.

### Содержимое scripts/

| Файл | Назначение |
|------|------------|
| **plot_wave_logs.py** | Парсит сводную таблицу волн (из лога или встроенные TABLE_ROWS), строит PNG-графики (длительность, путь, чекпоинты, руда, дашборд). Запуск: `python plot_wave_logs.py` или `python plot_wave_logs.py wave_log.txt`. |
| **wave_log.txt** | Пример файла с логом (таб-разделённые строки: первая — заголовок TABLE_HEADER, далее — данные волн). Можно копировать блок **[Сводка волн для графика]** из консоли игры в файл и передать скрипту. |
| **balance_tiers.py** | Скрипт баланса по ярусам крафта: перезаписывает урон в towers.json (0: −15%, 1: +7%, 2: +10%). См. [21-data-json](21-data-json.md). |

После запуска plot_wave_logs.py в текущей директории появляются PNG: wave_duration_and_size.png, wave_size_to30.png, checkpoint_progress.png, path_progress.png, path_length_hexes.png, segment_times.png, wave_success.png, tower_ore_per_round.png (если есть данные руды по вышкам), checkpoint_heatmap.png, ore_metrics.png, ore_by_sector.png, wave_dashboard.png.

### Интерактивные графики (wave_charts.html)

**Файл:** `docs/wave_charts.html`

HTML-страница с Chart.js для интерактивного просмотра тех же данных. Данные вставляются в скрипт в виде массива строк (каждая строка — одна волна, колонки в том же порядке, что и в сводной таблице лога). Подходит для быстрого просмотра одного прогона без установки Python: скопировать числа из лога, вставить в waveData в HTML, открыть файл в браузере.

Графики на странице: длительность волны и трата руды; путь (макс/ср %); руда (трата, добыча, накоплено); чекпоинты (до 0…5); время между чекпоинтами; HP и уровень игрока; урон вышек за волну 40 и общий урон по раундам.

### Цепочка работы с логами

1. Игра: после каждой волны в консоль выводится блок **[Сводка волн для графика]** (таб-разделённые колонки).
2. Скопировать заголовок и строки в файл (например scripts/wave_log.txt) или вставить в TABLE_ROWS в plot_wave_logs.py.
3. **Вариант A:** запустить `python scripts/plot_wave_logs.py scripts/wave_log.txt` — получить набор PNG в scripts/.
4. **Вариант B:** вставить данные в массив waveData в docs/wave_charts.html и открыть HTML в браузере — интерактивные графики.

Оба варианта используют один и тот же формат данных (сводная таблица из log_wave_damage_report).

## Snapshot при проигрыше

В начале каждой волны **GameManager.record_wave_snapshot()** добавляет в **game_state["wave_snapshots"]** снимок состояния: seed прогона, номер волны, список вышек по гексам. При первом показе экрана проигрыша вызывается **log_snapshot_on_game_over()** — в консоль выводится полный массив wave_snapshots в формате JSON. Это позволяет воспроизводить прогоны и использовать данные для симуляций. Инициализация wave_snapshots = [] выполняется в ECSWorld.init_game_state(). Подробнее: docs/23-game-manager.md (раздел «Snapshot»).
