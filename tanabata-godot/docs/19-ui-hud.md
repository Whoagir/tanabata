# UI и HUD

## Элементы

**InfoPanel (панель информации о выбранной сущности):**
- Показывается при выборе башни, врага или жилы руды. Содержит название, описание, скиллы, кнопки (Выбрать, Крафт и т.д.).
- **Переключатель вкл/выкл башни** — в заголовке панели, 50 px правее названия башни. Двухпозиционный слайдер в стиле «замок в самолёте»: влево = выкл (красный, приглушённый), вправо = вкл (зелёный, приглушённый). Отображается только для башен (не для стены). По клику или перетаскиванию переключается состояние башни (is_manually_disabled); то же действие доступно по ПКМ по башне в фазах WAVE или TOWER_SELECTION. Выключенная башня не стреляет и не тратит руду, но остаётся в сети (передаёт топологию); майнер при выключении только передаёт сеть, не добывает и не списывает руду с жилы под собой.
- **Батарея (TOWER_BATTERY):** у батареи тот же слайдер интерпретируется как **Добыча / Трата** (battery_manual_discharge: влево = Трата, вправо = Добыча). Ниже заголовка выводится блок информации о батарее (_add_battery_info): запас (battery_storage / storage_max), текущий режим. Обновление блока откладывается на следующий кадр (_deferred_battery_rebuild, таймер 0), чтобы не тормозить при переключении режима.

**Верхняя панель (слева):**
- Здоровье игрока (цвет по порогам: зелёный/жёлтый/красный).
- Номер волны.
- Счётчик башен (построено/лимит).
- Счётчик врагов: убито за всё время и живых сейчас / размер волны (total_enemies_killed, alive_enemies_count, размер волны из game_state).
- Уровень и XP (Lv.N, прогресс до следующего уровня).
- Кнопка «Рецепты (B)» — открывает книгу рецептов (таблица крафта).

**Топ-5 урона вышек (слева под панелью):**
- Во время волны — урон за текущую волну; в BUILD/SELECTION — за прошлую (last_wave_tower_damage).
- Строки вида «N. Имя MVP K: урон» (MVP показывается только при K > 0).
- Красная подсветка (RichTextLabel BBCode): строка топ-1 по урону и строки вышек с MVP 5.

**Блок пути и успеха (справа сверху):**
- Над индикатором руды: **длина пути** в три строки — «N гексов» (центр), подписи сегментов (в–1, 1–2, … 6–в), значения под столбцами; при менее чем 7 сегментах показывается только «N гексов». Ниже — **уровень успеха**: «Успех: N (M%)» (get_success_level, get_success_scale). Позиции блока и баров руды заданы с отступом от правого края (Config.SCREEN_WIDTH − 17 − ORE_WIDTH).

**Индикатор руды (справа сверху):**
- По одному прогресс-бару на каждую **сеть** (сеть = связная компонента энергосети, в которой есть руда: майнер на руде или батарея). Данные из EnergyNetworkSystem.get_networks_ore_and_attack_count(); показываются только сети с total_max > 0. При старте без вышек баров нет.
- Сети отсортированы по total_max (сверху — с наибольшим запасом руды). Под каждым баром подпись «X / Y» (current/max для этой сети). Расстояние между блоками (бар + подпись) увеличено на 8 px (ORE_BAR_GAP 24 px с учётом строки подписи).
- Цвет бара: синий при руде в сети >= 10; красный при руде < 10; при руде < 3 (ORE_FLICKER_THRESHOLD) — красное мигание.
- При наведении на майнер в InfoPanel показывается руда только его сети (get_network_ore_stats).

**Индикатор фазы (справа сверху):**
- Круг, цвет по фазе (синий/жёлтый/красный).
- В обычном режиме индикатор **не кликабелен**. Клик по нему переключает фазу только в **режиме разработчика** (клавиша **I**).

**Стеш (очередь слотов, слева снизу):**
- Виден только в фазе **BUILD**. Показывает оставшиеся слоты для постановки башен — **game_state["stash_queue"]** (буквы Б и А через пробелы). При постановке башни из очереди забирается один слот (тот тип, что ставится); при снятии майнера или батареи (Б) слот возвращается в начало очереди; при снятии атакующей вышки (А) слот не возвращается. Позиция: x=10, y=SCREEN_HEIGHT−40. Шрифт 17 pt, жирный, между буквами увеличенные пробелы. Б — жёлтый (#e6c230), А — серый (#a6a6b0). Начальное заполнение очереди при входе в BUILD — через GameManager.get_initial_stash_letters() (те же правила, что выбор типа при клике).

**Кнопки (справа снизу):**
- Скорость времени (1x, 2x, 4x). Цвет кнопки меняется.
- Пауза.

**Редактор энерголиний:** клавиша **U** переключает режим редактирования линий (line_edit_mode). Включённый режим отображается в HUD (например иконка «молния»). В этом режиме клики по карте передаются в LineDragHandler: перенаправление энерголинии от одной башни к другой (см. [06-systems-input](06-systems-input.md)#редактор-энерголиний-режим-u).

## Обновление

Счётчики (здоровье, волна, башни, убийства, живые враги), топ-5 урона и индикатор руды обновляются не каждый кадр, а с троттлом **до 4 раз в секунду** (интервал 0.25 с) для снижения нагрузки. Счётчик живых врагов берётся из **game_state["alive_enemies_count"]**. FPS обновляется каждый кадр. Индикатор фазы перерисовывается при смене фазы.

## Переходы фаз по клику

- BUILD → SELECTION: переход в выбор башен.
- SELECTION → WAVE: финализация выбора (невыбранные → стены), переход в волну.
- WAVE → BUILD: сброс счётчика, переход в строительство.

## Меню конца игры (Game Over)

При здоровье игрока 0 или ниже (в обычном режиме, не в режиме разработчика) игра заканчивается и показывается **модальное меню** (Popup):
- Заголовок «Игра окончена».
- Текст: «Вы набрали N очков», «Убито врагов: N».
- Кнопки: **«Начать заново»** (рестарт партии), **«В меню»** (выход в главное меню).
- Фон затемнённый, цвет как в главном меню (тёмно-синий/серый). Ввод под меню не проходит (панель руды, индикатор фазы и т.д. не получают клики).
- Реализация: `godot_adapter/ui/game_over_overlay.gd` (Popup), создаётся в GameRoot на отдельном CanvasLayer.

## Область кликов

Панель сверху не блокирует клики по карте (mouse_filter IGNORE). Кнопки индикатора и скорости перехватывают клики.
