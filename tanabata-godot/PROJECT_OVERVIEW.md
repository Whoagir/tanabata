# Tanabata (Godot) — краткое описание проекта

## О чём игра

**Tanabata** — 2D Tower Defense на гексагональной карте. Портируется с версии на Go. Игрок в фазе строительства ставит до 5 башен, в фазе выбора отмечает 2 башни для сохранения (остальные превращаются в стены), затем отражает волну врагов. Цикл повторяется: снова строительство, выбор, волна.

## Что есть в проекте

- **Карта:** гексы (pointy-top), процедурная генерация с входом, выходом и 6 чекпоинтами. Путь врагов: A* через чекпоинты (дебаунс пересчёта 0.08 с); для летающих — условная вставка центра между чекпоинтами.
- **Башни:** майнеры (дают энергию с руды), атакующие (стреляют по врагам, питаются от энергосети), стены (блокируют путь). Вкл/выкл вручную (ПКМ или слайдер в InfoPanel); выключенный майнер только передаёт сеть.
- **Энергосеть:** майнеры на руде — источники; атакующие работают только при соединении с сетью (MST, перехват майнер-майнер на линии). Редактор линий — клавиша **U** (перенаправление связи между башнями).
- **Руда:** генерируется при старте (3 жилы), расходуется при выстрелах; восстановление за волну фиксированное (до 20 тиков). Ниже порога гекс не питает майнер. HUD: прогресс-бар «Руда: X / Y», при наведении на майнер — руда его сети.
- **Враги:** спавн по таймеру от входа, движение по пути, урон игроку при достижении выхода; броня (физ/маг), уклонение (evasion), способности из ability_definitions.json. Статус-эффекты: slow, poison, jade_poison (стакающийся).
- **Фазы:** BUILD (до 5 башен) → TOWER_SELECTION (выбор 2, остальные в стены) → WAVE (враги и бой) → снова BUILD. Завершение волны — по счётчику живых врагов (alive_enemies_count).
- **Крафт:** соседние башни по рецептам из recipes.json превращаются в новые типы (Silver, Malachite, Jade, Grey, Bloodstone, Antique и др.); стоимость в руде по ярусу. Recipe Book (B) — таблица всех рецептов.
- **MVP:** у каждой башни уровень 0–5 (+20% урона за уровень); в конце волны +1 MVP первой по урону (если волна не пропущена).
- **Босс и карты:** при убийстве босса (ENEMY_BOSS) — оверлей с тремя картами (две благословения, одна проклятие); выбор применяет эффект (CardsData).
- **UI:** HUD (здоровье, волна, башни, убийства, живые враги, топ-5 урона, «Рецепты (B)», индикатор фазы, скорость 1x/2x/4x, пауза), InfoPanel (инфо, «Сохранить», слайдер вкл/выкл башни). При HP <= 0 — меню конца игры (очки, «Начать заново» / «В меню»). Клавиша **I** — режим разработчика (кликабельный индикатор фазы, HP <= 0 не завершает игру).
- **Уровни и туториал:** LevelConfig задаёт параметры карты и пошаговые подсказки; уровни обучения 0–4 (Основы, Энергия и руда, Стены и выбор, Крафт, Практика).
- **Логи и аналитика:** после каждой волны в консоль выводится отчёт (чекпоинты, путь, руда по секторам, основная сеть %, урон вышек) и сводная таблица для графиков; скрипт `scripts/plot_wave_logs.py` строит графики по сводке.

## Что происходит по шагам

1. **Старт:** GameManager создаёт ECS, HexMap, загружает JSON (башни, враги, волны, лут), генерирует руду, создаёт EnergyNetworkSystem. Main загружает GameRoot; GameRoot создаёт системы и рендереры, рисует карту.
2. **BUILD:** игрок кликает по гексу — ставится башня (тип по волне/луту или дебаг-клавишам). Майнеры на руде и соседние башни связываются энергосетью. После 5 башен фаза переключается в SELECTION.
3. **SELECTION:** игрок отмечает до 2 башен кнопкой «Сохранить» в InfoPanel или кликом. Майнеры считаются выбранными по умолчанию. Клик по индикатору фазы или выбор второй башни запускает переход в WAVE: невыбранные временные башни удаляются и заменяются стенами, энергосеть пересобирается.
4. **WAVE:** WaveSystem создаёт сущность волны, считает путь через чекпоинты, спавнит врагов по таймеру. MovementSystem двигает врагов по пути; при достижении выхода — урон игроку, сущность удаляется. CombatSystem ищет цели, создаёт снаряды/лазеры, списывает энергию с руды. ProjectileSystem двигает снаряды, при попадании наносит урон (броня, статусы). StatusEffectSystem обновляет таймеры slow/poison. Когда все враги заспавнены и нет живых (убиты или дошли до выхода) — волна завершена, фаза переключается в BUILD.
5. **Рендеринг:** EntityRenderer (башни, враги, снаряды, лазеры, цепная молния, подсветка гексов, линии майнер–стена), WallRenderer (стены и связи), OreRenderer, EnergyLineRenderer, AuraRenderer, TowerPreview, PathOverlayLayer, AttackLinkRenderer, CraftingVisualRenderer. Данные берутся из ECS и HexMap.

## Крафтованные и спецбашни (кратко)

- **Silver** — лазер, замедление. **Malachite** — сплит + impact burst, осколки с homing. **Volcano** — AoE тики по площади (8 тиков/с). **Lighthouse** — вращающийся луч (Beacon). **Jade** — стакающийся яд (JADE_POISON). **Grey** — Jade + цепной яд, усиленный урон.
- **Ruby** — тиковый маг. урон (8 тиков/с), дебафф маг. брони −0.2 за попадание. **Bloodstone** — снаряд 100 маг. урона; 17% шанс цепной молнии по 5 целям (950 урона), белые линии с изломами, следуют за врагами.
- **Antique** (Рубин + Бладстоун + TO3) — тиковый урон 250 маг., дальность 4 гекса, дебафф маг. брони как Рубин; 2% шанс за тик — цепная молния на 7 целей по 4000 урона (визуал жирнее и дольше).
- **Egypt** — снаряд 33 PURE, дебафф брони −25, замедление 8%; 20% шанс при попадании дать +5 руды в сеть.
- **Pink** — лазер 890 PURE, крит 35% ×2.6. **Huge** — лазер 3900 SPLIT (50% физ/50% пуре), крит 10% ×9, сплеш при крите. **Auriga** — линия урона до 5 гексов, поворот по направлению с макс. врагов.

## Технологии и структура

- **Core (без Godot):** ECS, Hex/HexMap/Pathfinding, системы (input, wave, movement, combat, projectile, status_effect, aura, crafting, volcano, beacon, auriga, line_drag_handler), energy_network, ore_generation. Данные — через DataRepository (towers, enemies, waves, recipes, loot_tables, ability_definitions).
- **Godot adapter:** рендереры в `godot_adapter/rendering/`, UI в `godot_adapter/ui/` (HUD, InfoPanel, Recipe Book, BossRewardOverlay, GameOverOverlay). Ввод обрабатывается в GameRoot и передаётся в InputSystem (очередь команд).
- **Autoload:** Config (константы, цвета, load_json), DataRepository (загрузка JSON), GameManager (ECS, карта, энергосеть, API), Profiler (опционально).
